## 评审检查清单

完成代码生成或修改后，逐项确认以下内容：

- [ ] 响应结构使用统一 `CommonResp` 封包（`code` / `success` / `data` / `message`）
- [ ] API 层对失败路径使用 `types.Error(err)` 返回，处理器保持薄层
- [ ] RPC 层对预期失败路径返回 `pkg/err` 类型化错误
- [ ] 日志携带稳定标识字段，无敏感信息泄露（密码、token 明文、验证码）
- [ ] 数据库操作条件显式、错误映射完整、跨存储操作有回滚或补偿策略说明
- [ ] 新增中间件具备明确的职责、顺序决策依据与公开接口的 bypass 策略
