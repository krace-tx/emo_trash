# Go 服务开发规范 Skill

## 适用场景

在以下目录或模块进行开发时，必须应用本 Skill：

- `app/api/gateway/**`：HTTP 网关 API 处理器与逻辑层
- `app/rpc/sso/**`：SSO 认证 RPC proto、server 与 logic
- 共享包：`pkg/datastore`、`pkg/err`、`pkg/page`
- 中间件、响应封装器、认证/登录流程

## 项目基线规则

以下规则适用于所有代码生成与修改，不得例外：

- **统一响应结构**：API 层使用 `types.CommonResp` 作为统一响应封包（字段：`code` / `success` / `data` / `message`）。
- **API 层薄层原则**：API 逻辑通过 `types.Error(err)` 转换错误，禁止在 API 层嵌入业务决策。
- **RPC 层业务内聚**：RPC 逻辑负责参数校验、状态检查、token/session 流程以及多存储编排。
- **类型化错误**：已知业务失败场景返回 `pkg/err` 定义的类型化错误，禁止透传原始 error。
- **数据库访问封装**：SQL 操作使用 `sqlstore` 引擎封装，Mongo 通过 `svcCtx` 注入客户端，保持查询意图显式。
- **日志字段稳定性**：日志必须携带可用的稳定标识字段：`user_id`、`account`、`platform`、`device_id`、`ip`；禁止记录敏感信息（密码、token 明文、验证码）。

## Agent 代码输出要求

生成或修改代码时，须满足以下约束：

- 命名与包结构与仓库现有 go-zero 风格保持一致。
- 优先使用短小聚焦的方法，重复逻辑提取为私有 helper 函数。
- 注释仅保留非显而易见的业务规则或副作用说明，不写重复语义的注释。
- 不得破坏现有 API 字段名与响应结构的向后兼容性（有明确需求除外）。
